Project FAQ

These notes explain the constraints and assumptions behind the public writing and reading experience.

Why are posts ASCII-only? Canonical text records stay easier for humans to read, patch, compare, and review across ordinary tools, while reducing Unicode lookalike obfuscation risks.
What is stored here? Threads, replies, moderation records, and public instance metadata are stored as canonical text records tracked in git.
What is this page for? It combines project orientation with the current public instance facts so readers can understand both the system and this deployment.
What is not the goal? This is not a generic social feed or a rich-text publishing stack; the project favors plain text, inspectable records, and deterministic history.
