Manual signed profile update

You can complete this without the local browser key flow. Use any signer that can sign the exact payload as ASCII text, produce an ASCII-armored detached signature, and export the matching ASCII-armored public key.

  1. Enter the desired display name so the page can derive the canonical payload preview below.
  2. Copy the exact text from Canonical payload preview.
  3. Sign that exact payload as a detached ASCII-armored signature using the key for ${source_identity_id}.
  4. Export the matching ASCII-armored public key.
  5. Submit one JSON request to /api/update_profile with payload, signature, public_key, and dry_run.

Example request body:

{
  "payload": "...exact canonical payload text...",
  "signature": "-----BEGIN PGP SIGNATURE-----\n...\n-----END PGP SIGNATURE-----\n",
  "public_key": "-----BEGIN PGP PUBLIC KEY BLOCK-----\n...\n-----END PGP PUBLIC KEY BLOCK-----\n",
  "dry_run": false
}

The signing key must resolve to ${source_identity_id} or the server will reject the update.
